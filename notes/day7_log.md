### 一、机器学习基础复习

今天重点复习了机器学习中的几个核心概念：

- **监督学习（Supervised Learning）**  
  利用“输入 + 标签”的数据进行训练，使模型学会从输入预测对应输出。在我的项目中，Router 分类器就是典型的监督学习问题：输入是用户问题，标签是问题类型。

- **无监督学习（Unsupervised Learning）**  
  没有标签，模型需要自己从数据中发现结构，比如聚类。但在当前项目中，Router 不属于无监督学习，而是明确的监督学习任务。

- **损失函数（Loss Function）**  
  用来衡量模型预测结果与真实标签之间的差距。训练的目标就是不断降低损失函数，使模型预测更准确。

- **梯度下降（Gradient Descent）**  
  一种优化方法，通过不断调整模型参数，使损失函数最小化。虽然本项目使用的是现成模型（如 SVM），但本质仍属于优化问题。

### 二、什么是分类任务（Classification）

分类任务是机器学习中最基本的问题之一：

> 给定输入数据，将其划分到若干预定义类别中。

在本项目中：

- 输入：用户提出的问题（自然语言文本）
- 输出：问题类别（concept_explanation  / web_search）

例如：

- “什么是 SQL 注入？” → concept_explanation  
- “我想学 Web 安全” → study_advice  

因此，我的 Router 本质上就是一个**文本分类器（Text Classifier）**。

---

### 三、文本分类与 Router 的关系

Router 的作用是：

> 判断用户问题属于哪一类，并将其分发到对应模块处理。

这与文本分类完全一致：

| 模块功能 | 对应标签 |
|----------|----------|
| 概念解释 | concept_explanation |
| 学习建议 | study_advice |
| 联网搜索 | web_search |

也就是说：

> Router ≈ 文本分类器  
> Agent 的“决策能力”本质上就是一个分类问题

这让我意识到，智能体系统并不是完全全新的东西，而是建立在机器学习基础之上的组合系统。

---

### 四、为什么 Router 可以“先规则后模型”

在本项目中，我采用了三阶段 Router 设计：

#### 1）规则路由（Rule-based Router）

使用 if-else 判断关键词：

- “什么是 / 解释” → 概念解释
- “根据资料” → RAG
- “怎么学” → 学习建议

优点：
- 简单、快速实现  
缺点：
- 覆盖不全
- 易冲突
- 泛化能力差

---

#### 2）LLM 路由（大模型判断）

让大语言模型判断问题类型。

优点：
- 语义理解能力强  
缺点：
- 成本高
- 输出不稳定
- 不可控

---

#### 3）机器学习路由（最终方案）

使用 **TF-IDF + SVM** 训练分类器：

流程：
- 构造数据集（问题 + 标签）
- 文本向量化（TF-IDF）
- 训练分类模型（SVM）
- 预测问题类别

优点：
- 稳定
- 高效
- 可解释
- 符合已有机器学习基础
