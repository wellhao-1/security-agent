# Day 8–Day 14 学习日志补写

这几天我主要在补大模型和智能体相关的基础知识。因为我一开始只学过机器学习，对深度学习、NLP、Transformer、LLM、RAG 和 Agent 这些内容还不是很熟，所以这一周的重点不是马上写代码，而是先把这些概念串起来，理解它们和我的“信息安全学习助手 Agent”项目有什么关系。

## Day 8

今天主要学了深度学习和 Embedding。

我之前对深度学习的理解比较模糊，只知道它和神经网络有关。今天重新梳理了一下，深度学习其实就是通过多层神经网络自动学习数据里的特征。传统机器学习很多时候需要人工设计特征，而深度学习可以自己从数据中学习更复杂的表示。

今天最重要的概念是 Embedding，也就是“嵌入表示”。它的作用是把文字、句子或者文档变成向量。因为模型不能直接处理自然语言文本，所以必须先把文字变成数字，模型才能进行计算。

这和我的项目关系很大。后面做 RAG 的时候，需要把用户问题和本地知识库里的文档都变成向量，然后比较它们之间的相似度，找到最相关的内容。虽然我后面 Router 分类器准备用 TF-IDF + SVM，不是深度学习 Embedding，但本质上也是把文本变成数字特征，再交给模型分类。

今天最大的收获是，我开始理解为什么“文本表示”这么重要。无论是 RAG 检索，还是 Router 分类，第一步都是把自然语言转成模型能处理的形式。

## Day 9

今天学的是 NLP 基础。

我主要理解了 NLP、Token、Tokenization、Corpus、Vocabulary、文本分类和句子表示这些概念。

以前我会以为模型是直接理解一句完整的话，但今天学完之后发现不是这样。大模型处理文本时，是先把句子拆成一个个 token，再对 token 进行处理。比如“什么是 SQL 注入？”这句话，可以理解成由“什么”“是”“SQL”“注入”等 token 组成。

今天我还重点理解了文本分类。文本分类就是输入一段文本，然后判断它属于哪个类别。这个概念和我的 Router 模块直接相关。

我的项目里，Router 要判断用户问题属于哪一类，比如：

- “什么是 SQL 注入？”属于 concept_explanation
- “根据资料解释 HTTPS 的流程”属于 rag_qa
- “我想学 Web 安全，应该怎么开始？”属于 study_advice
- “最近有哪些高危漏洞？”属于 web_search

所以我现在更清楚了：Router 本质上就是一个文本分类器。它不是一个很玄的模块，而是可以用机器学习方法来实现的。

今天的收获是，我把 NLP 里的文本分类和我的项目真正联系起来了。后面训练 Router 分类器，其实就是在做一个小型文本分类任务。

## Day 10

今天学了 Attention、Self-Attention 和 Transformer。

刚开始听 Transformer 会觉得很复杂，但今天我先没有去推公式，而是先理解它为什么有用。Attention 的核心意思就是：模型在理解一个词的时候，会去看上下文里哪些词更重要。

比如“SQL 注入是一种 Web 攻击”，模型在理解“攻击”的时候，应该重点关注“SQL 注入”和“Web”。如果是“他把书放到桌子上，因为它太重了”，模型要判断“它”指的是“书”，也需要看上下文。

Self-Attention 就是句子内部的 token 之间互相关注，计算彼此之间的关系。Transformer 之所以适合处理文本，就是因为它可以很好地建模 token 和 token 之间的关系。

今天还学了 Context Window，也就是上下文窗口。这个概念也和我的项目有关。因为 LLM 一次能处理的文本长度是有限的，所以不能把所有资料都直接塞给模型。RAG 就是先检索最相关的内容，再把少量有用内容交给 LLM，这样更合理。

今天的收获是，我知道了现代大模型为什么大多基于 Transformer。虽然我不会从零训练 Transformer，但理解 Attention 和 Transformer，可以帮我理解 LLM 为什么能处理自然语言，也能理解后面 RAG 和 Prompt 为什么重要。

## Day 11

今天学了 LLM 的训练流程。

我主要了解了 Pretraining、Fine-tuning、SFT、RLHF 和 Hallucination。

预训练就是让模型在大量文本上学习语言规律，主要任务是预测下一个 token。这个阶段模型可以说是“会补全句子”，但还不一定真的会按照人的要求回答问题。

微调是在预训练模型的基础上，让模型更适合某些任务。SFT 是监督微调，用标注好的问答数据让模型学习如何回答。RLHF 是人类反馈强化学习，它的作用是让模型的回答更符合人类偏好，比如更有帮助、更安全、更像正常对话。

今天我还重点理解了 Hallucination，也就是幻觉。LLM 有时候会生成看起来很合理但实际上不正确的内容。这个问题和我的项目关系很大，因为如果我的信息安全学习助手直接靠 LLM 回答，可能会出现不准确甚至误导性的内容。所以需要 RAG，让模型基于本地知识库回答，减少胡编。

今天最大的收获是，我理解了 LLM 不是一开始就会对话的，而是经过预训练、微调和对齐之后，才变得适合实际使用。同时我也知道了 LLM 有局限，不能完全相信它自己的“记忆”，所以需要 RAG 来增强可靠性。

## Day 12

今天学的是 Prompt Engineering。

我以前觉得 Prompt 就是“问模型一个问题”，但今天发现它其实是控制模型行为的重要方式。不同的提示词会影响模型回答的风格、结构和准确性。

今天我主要了解了 System Prompt、User Prompt、Zero-shot、Few-shot、Chain-of-Thought 和 ReAct 这些概念。

然后我开始考虑我的项目里需要哪些 Prompt。大概有几类：

第一类是概念解释 Prompt，用来回答“什么是 SQL 注入”“什么是 XSS”这种问题。

第二类是 RAG Prompt，用来把检索出来的资料和用户问题放在一起，让模型根据资料回答。

第三类是学习建议 Prompt，用来回答“Web 安全怎么学”“密码学怎么入门”这种问题。

第四类是 Router Prompt。因为我的自训练 Router 分类器要到后面才训练出来，所以在前期可以先让 LLM 临时帮我判断问题类别。比如让它只输出 concept_explanation、rag_qa、study_advice 或 web_search 其中一个标签。

今天的收获是，我意识到 Prompt 不是随便写的，而是项目里很重要的一部分。好的 Prompt 可以让 LLM 更稳定地完成任务，也可以在 Router 分类器没有完成之前当作临时方案。

## Day 13

今天学了 RAG，也就是检索增强生成。

RAG 的思路是：不要完全依赖大模型自己的知识，而是先从外部知识库里检索相关资料，再让模型根据这些资料生成回答。

我理解的流程是：

先整理本地知识库，比如 SQL 注入、XSS、CSRF、HTTPS、哈希、加密这些信息安全基础资料。然后把文档切成小块，再把这些文本块转换成向量。用户提问之后，系统也把用户问题转换成向量，然后去知识库里找最相关的几个文本块。最后，把这些检索结果和用户问题一起放进 Prompt，让 LLM 根据资料回答。

这和我的项目关系非常直接。我的信息安全学习助手不能只是一个普通聊天机器人，而应该能基于我整理的本地资料回答问题。比如用户问“根据资料，HTTPS 的工作流程是什么？”，系统应该先去知识库里找 HTTPS 相关内容，再生成回答。

今天我也理解了为什么 RAG 能减少幻觉。因为它给 LLM 提供了依据，模型不需要完全凭自己的参数记忆回答。

今天的收获是，我终于把 RAG 的流程想清楚了：文档收集、文档切块、文本向量化、相似度检索、Prompt 拼接、LLM 生成。后面写项目代码时，Retriever 模块就是围绕这个流程来做。

## Day 14

今天主要整理了整个项目前期方案。

我现在把自己的项目定位成“信息安全学习助手 Agent”，它不是一个单纯的聊天机器人，而是一个模块化的小型 Agent。

我目前理解的系统结构是这样的：

用户输入问题之后，Main Agent 先接收问题，然后交给 Router 判断问题类型。Router 判断之后，再把问题交给对应模块：

- 如果是概念解释，就让 LLM 解释安全概念
- 如果是资料问答，就走 RAG，先检索本地知识库再回答
- 如果是学习规划，就交给 Advisor 生成学习路线
- 如果是最新安全事件，就交给 web_search 处理
- 回答结束后，再用 Memory 保存最近几轮对话

今天我也重新明确了 Router 的升级路线：

一开始先用规则路由，也就是 if-else 和关键词判断。比如问题里有“什么是”“解释”，就分到概念解释；有“根据资料”“知识库”，就分到 RAG；有“怎么学”“入门”，就分到学习建议。

第二步可以用 LLM 路由，也就是让大模型临时判断问题类别。

最后再用自己构造的数据集训练 TF-IDF + SVM 分类器，把 Router 升级成一个真正的自训练文本分类模块。

我觉得这个设计是目前项目里比较有亮点的地方。因为它说明我不是只会调用 API，而是把机器学习里的文本分类方法接进了 Agent 系统。

今天的收获是，我对整个项目的结构更清楚了。这个项目后面主要会包括 Main Agent、Router、Router Classifier、Retriever、LLM、Advisor 和 Memory 这些模块。接下来就要从理论学习转到代码实现了。

## 这几天的整体总结

这几天我最大的变化是：一开始我只是觉得“智能体”是一个很热门但有点抽象的方向，现在我逐渐能把它拆成具体模块来看了。

我现在的理解是，Agent 不是单纯的大模型，而是一个系统：

- LLM 负责生成回答
- Router 负责判断问题走哪个模块
- Retriever 负责从本地知识库找资料
- RAG 负责让回答更有依据
- Advisor 负责生成学习路线
- Memory 负责保存最近对话
- Main Agent 负责把这些模块组织起来

这几天我也慢慢把机器学习基础和智能体项目联系起来了。特别是 Router 这个模块，我现在理解它本质上就是文本分类。先用规则实现，再用 LLM 过渡，最后用 TF-IDF + SVM 训练一个轻量分类器，这条路线比较适合我现在的基础。
